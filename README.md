# “Qual a importância do tratamento de exceções em um sistema orientado a objetos?”

O tratamento de exceções separa a lógica de erro da lógica principal (caminho feliz), tornando o código mais limpo.
Permite que objetos protejam seu estado interno (encapsulamento) ao se recusarem a executar operações inválidas.
Garante software robusto ao propagar erros de forma controlada até o ponto certo para tratá-los, evitando que o sistema quebre.
